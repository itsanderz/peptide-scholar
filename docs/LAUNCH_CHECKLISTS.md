# Launch Checklists

Date: April 11, 2026
Purpose: Prevent low-quality launches by defining the minimum release checks for pages, markets, tools, partners, pSEO batches, and the PWA.

## 1. Page Family Launch Checklist

- [ ] Schema finalized and approved
- [ ] Renderer exists and matches the schema
- [ ] Primary CTA defined
- [ ] CTA behavior is market-aware
- [ ] Canonical logic defined
- [ ] Internal links defined
- [ ] FAQ and structured data validated
- [ ] Disclaimer present
- [ ] Trust block present
- [ ] Source list complete
- [ ] Mobile QA complete
- [ ] Analytics events firing
- [ ] Clarity session test recorded
- [ ] Noindex/index rule confirmed
- [ ] Editorial review state assigned

## 2. Market Launch Checklist

- [ ] Market schema exists
- [ ] Default locale defined
- [ ] Supported locales defined
- [ ] Country-specific product availability reviewed
- [ ] Country-specific pricing context reviewed
- [ ] Country-specific legal framing reviewed
- [ ] Provider routing state defined
- [ ] Unsupported fallback CTA defined
- [ ] Disclosure copy reviewed
- [ ] Currency and measurement rules implemented
- [ ] Search monitoring plan in place
- [ ] CRM routing for leads or waitlists configured
- [ ] Country selector UX tested
- [ ] Market pages QAed on mobile and desktop
- [ ] Launch owner assigned

## 3. Tool Launch Checklist

- [ ] Tool spec signed off
- [ ] Inputs and formulas reviewed
- [ ] Output language reviewed
- [ ] Disclaimer shown at the right moments
- [ ] Example states or sample outputs present
- [ ] Save and export behavior defined
- [ ] App handoff defined
- [ ] Analytics events for start, completion, export, and save implemented
- [ ] Empty, invalid, and edge states tested
- [ ] Mobile keyboard and layout QA complete
- [ ] SEO landing page copy reviewed
- [ ] Tool page CTA reviewed

## 4. Provider / Partner Launch Checklist

- [ ] Partner verified
- [ ] Commercial terms documented
- [ ] Lead-routing method tested
- [ ] Disclosure language approved
- [ ] Claims about provider services reviewed
- [ ] Market allows provider CTA
- [ ] Fallback path exists if partner is unavailable
- [ ] Attribution and payout logic tested
- [ ] Lead-quality QA plan in place
- [ ] Internal owner for partner health assigned

## 5. pSEO Batch Launch Checklist

- [ ] Taxonomy subset finalized
- [ ] Target query cluster validated
- [ ] Schema fields complete
- [ ] Deterministic fields generated
- [ ] Source-backed fields populated
- [ ] Review state assigned per page
- [ ] Index vs noindex decision made
- [ ] Internal link paths assigned
- [ ] Template QA completed on sample pages
- [ ] Analytics coverage verified on sample pages
- [ ] Batch size approved
- [ ] Post-launch indexation review date scheduled

## 6. PWA Beta Checklist

- [ ] Core use case defined
- [ ] Local-first storage behavior tested
- [ ] Reminder behavior tested
- [ ] Export flow tested
- [ ] Privacy and disclaimer copy finalized
- [ ] Waitlist-to-beta invite flow ready
- [ ] Premium boundary defined
- [ ] Support and feedback channel defined
- [ ] D7 and D30 retention dashboards ready
- [ ] Beta invite cohort defined
- [ ] Crash and error logging verified

## 7. Launch Stop Conditions

- Any market page with unreviewed regulatory or product claims
- Any provider launch without routing and attribution QA
- Any tool with unreliable calculations or misleading outputs
- Any pSEO batch where sample pages fail the bookmarkability test
- Any PWA beta without analytics, error monitoring, and export validation
